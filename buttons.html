Buttons

A button is an interactive element that enables users to get the expected interactive feedback from the web interface or mobile application following a particular command. Basically, a button is a control that allows a user to communicate directly with the digital product and send the necessary commands to achieve a particular goal. For example, it may be the task to send an email, buy a product, download some data or a piece of content etc.

In mobile accessibility testing, ensuring buttons are fully perceivable and operable is critical because a broken button completely blocks a user's journey. When auditing buttons, the rules and behaviors shift depending on whether you are testing a Native App or a Mobile Web App.

Native vs. Mobile Web Buttons
  • Native Apps: Built using standard platform components, these automatically announce their role (e.g., "Save, Button"). Bugs happen when developers build custom buttons out of flat text or images without labeling them as interactive elements.
  • Mobile Web Apps : Run inside mobile browsers using web tags. Standard HTML buttons work automatically, but problems arise when generic tags (like div or span) are styled to look like buttons without adding proper accessibility roles.
  • Verify the Label: Every button must announce a clear, descriptive purpose. Icon-only buttons (like a trash can icon) must say something meaningful like "Delete item" instead of just "Button" or remaining completely silent.
  • Confirm the "Button" Role: Listen for the word "Button" at the end of the announcement (e.g., "Submit, Button"). If it is missing, the element lacks the correct structural role, and users won't know it can be tapped.
  • Validate Touch Target Sizes: Ensure buttons meet minimum physical size guidelines so they are easy to tap without missing:
    • Android: Minimum area of 48 x 48 dp.
    • iOS: Minimum area of 44 x 44 pt.

Screen Reader Navigation Gestures for Buttons

Users can skip directly from button to button by changing their screen reader's navigation mode.

  1. Change Navigation Mode to Controls - Swipe up then down (or down then up) with a single finger until you hear TalkBack say "Controls". Alternatively, you can use a three-finger swipe up or down.
  2. To move to the Next Button - Swipe down with one finger.
  3. To move to the Previous Button - Swipe up with one finger.
  4. To activate the Button - Double-tap anywhere on the screen.

To move between buttons using VoiceOver, you use the Rotor, which acts as a virtual dial for navigation settings.

  1. Change Navigation Mode to Buttons - Rotate two fingers on the screen (like turning a physical dial) until you hear VoiceOver say "Buttons" or "Form Controls".
  2. To move to the Next Button - Swipe down with one finger.
  3. To move to the Previous Button - Swipe up with one finger.
  4. To activate the Button - Double-tap anywhere on the screen.
