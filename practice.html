Mobile Accessibility Testing Exercise

Now that we have reviewed our core mobile testing strategies over the previous pages, it is time for a quick, friendly exercise to put our knowledge into practice! Treat this page as a live sandbox and explore the elements below to see how closely theory aligns with the actual user experience. Let us see how many accessibility insights we can uncover together before jumping into the assessment!

Instructions for the Evaluation Exercise (Double tap to expand / collapsed)

  1. Thoroughly audit this page for screen reader-related accessibility defects using your mobile device's screen reader (TalkBack on Android or VoiceOver on iOS).
  2. Ensure your testing covers the entire page structure, starting from the main landmark/ heading level 1 all the way through to the final link present at the bottom of the page.
  3. Limit your evaluation parameters strictly to the accessibility criteria, semantic elements and mobile gestures covered in our previous training modules.
  4. Document or note down any semantic gaps, announcement errors or navigation bugs you discover, as you will need to reference them to answer the multiple-choice questions (MCQs) on the next page.
