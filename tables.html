Data Tables

Data tables are highly efficient for organizing complex information, but they present unique challenges when translated to mobile touch-screens. When performing an accessibility audit, it is critical to understand that data tables behave fundamentally differently depending on whether they are built into a Native App or a Mobile Web App.

Native mobile frameworks do not have a standard, built-in "HTML-style" grid system that automatically links column headers to data cells out of the box. Testing Experience: When testing a native app table, screen readers treat the grid as a flat, sequential list of text blocks. Because there is no native "Table Mode" gesture navigation for native components, you must test that the developers have grouped elements or manually added text labels so that every single data cell announces its context (e.g., instead of just reading "50", it should read "Remaining Leaves: 50").

Mobile web apps inherit the rich semantic structure of HTML tags (e.g., table, tr, th, td). Testing Experience: Mobile web browsers fully support table navigation modes. When a screen reader encounters a properly marked-up web table on a phone, the user can change their local settings to navigate purely by rows or tables, allowing them to jump through tabular data horizontally and vertically just like on a desktop.

  • Verify Contextual Continuity: As you swipe through data cells, the screen reader must convey the cell's relationship to its header.
    • On native apps, ensure the text description itself explicitly includes the header context.
    • On mobile web, verify the browser reads the column header automatically.
  • Test the Reading Flow Order: Swiping left-to-right should move the screen reader focus logically across a single row before proceeding to the next. If the focus jumps erratically down columns or out of order, the reading flow is broken.
  • Check for Table Mode : On mobile web, switch your screen reader to "Table" mode and ensure it recognizes the container. If the device emits an error sound or says "No tables found", the website is using visual styling (like CSS flexbox or divs) to mimic a table without using semantic HTML markup.

Screen Reader Navigation Gestures for Data Tables

To navigate tables on the mobile web, you can add "Tables" to your reading controls via TalkBack settings.

  1. Change Navigation Mode to Tables: Swipe up then down (or down then up) with a single finger until you hear TalkBack say "Tables". Alternatively, use a three-finger swipe up or down.
  2. To move to the Next row - Swipe down with one finger.
  3. To move to the Previous row - Swipe up with one finger.
  4. To move to the next column - Swipe right with one finger.
  5. To move to the previous column - Swipe left with one finger.

When a semantic table is active on a web page, VoiceOver unlocks structural options within the Rotor.

  1. To change Navigation Mode to rows/Tables rotate two fingers on the screen (like turning a physical dial) until you hear VoiceOver say "Rows" or "Tables".
  2. To move to the Next Row - Swipe down with one finger (bypasses individual cells to move straight down the grid).
  3. To move to the Previous Row - Swipe up with one finger.
  4. For navigating Inside a Row - Swipe right (next cell) or left (previous cell) with one finger.
