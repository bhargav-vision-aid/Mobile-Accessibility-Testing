About Headings

Headings are titles or subtitles that are displayed on a webpage or mobile screen. In HTML, headings are defined with the h1 to h6 tags. Screen reader Users often skim a page by its headings. It is important to use headings to show the document structure.

During an accessibility audit, testing headings ensures that visual titles match semantic reality. When a screen reader user switches their device's navigation mode to look specifically for headings, properly configured titles allow them to jump directly to major sections, bypassing repetitive content, images, and buttons entirely.

  • Both iOS and Android support headings across native, hybrid, and mobile web environments.
  • Native iOS apps have better support for headings as developers can define heading levels, just like in HTML – from level 1 to level 6. iOS also supports just a heading, with no level.
  • On the Android platform for native apps we can still make a semantic heading but we don’t have the possibility for different levels.
React Native supports headings but no levels.

Screen Reader Navigation Gestures for Headings

To move between headings using TalkBack, you must first change your reading control setting to "Headings."

  1. Change Navigation Mode to Headings - Swipe up then down (or down then up) with a single finger until you hear TalkBack say "Headings". Alternatively, you can use a three-finger swipe up or down.
  2. To move to the Next Heading - Swipe down with one finger.
  3. To move to the Previous Heading - Swipe up with one finger.

To move between headings using VoiceOver, you use the Rotor, which acts as a virtual dial for navigation settings.

  1. To change Navigation Mode to Headings rotate two fingers on the screen (like turning a physical dial) until you hear VoiceOver say "Headings".
  2. To move to the Next Heading - Swipe down with one finger.
  3. To move to the Previous Heading - Swipe up with one finger.
